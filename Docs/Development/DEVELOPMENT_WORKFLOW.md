# Development Workflow & Standards\n\n## Project: Vibe Dev Squad Official\n**Last Updated:** December 24, 2024\n\n---\n\n## Git Workflow & Branching Strategy\n\n### Branch Structure\n- **`main`** - Production-ready code, always stable\n- **`develop`** - Integration branch for features\n- **`feature/*`** - Feature development branches\n- **`fix/*`** - Bug fix branches\n- **`hotfix/*`** - Critical production fixes\n\n### Branching Rules\n1. **Never commit directly to `main`**\n2. **Create feature branches from `develop`**\n3. **Use descriptive branch names**:\n   - `feature/user-authentication`\n   - `fix/navigation-mobile-bug`\n   - `hotfix/security-vulnerability`\n\n### Workflow Process\n1. Create branch from `develop`\n2. Make changes and commit with conventional messages\n3. Push branch and create Pull Request\n4. Code review and testing\n5. Merge to `develop`\n6. Deploy to staging for testing\n7. Merge `develop` to `main` for production\n\n---\n\n## Commit Message Standards\n\n### Conventional Commits Format\n```\ntype(scope): subject\n\nbody (optional)\n\nfooter (optional)\n```\n\n### Commit Types\n- **`feat:`** - New feature\n- **`fix:`** - Bug fix\n- **`docs:`** - Documentation changes\n- **`style:`** - Code style changes (formatting, etc.)\n- **`refactor:`** - Code refactoring\n- **`test:`** - Adding or updating tests\n- **`chore:`** - Maintenance tasks\n- **`perf:`** - Performance improvements\n- **`ci:`** - CI/CD configuration changes\n\n### Examples\n```bash\n# Good commit messages\nfeat(auth): add OAuth login with GitHub\nfix(nav): resolve mobile menu overflow issue\ndocs(api): update endpoint documentation\ntest(components): add unit tests for Button component\nchore(deps): update dependencies to latest versions\n\n# Bad commit messages\nfixed stuff\nupdate\nworking on feature\nfhsadlgkjadsjkl\n```\n\n### Commit Message Rules\n- **Subject line**: 50 characters or less\n- **Capitalize** the subject line\n- **No period** at the end of subject\n- **Use imperative mood** (\"add\", not \"added\")\n- **Explain what and why**, not how\n\n---\n\n## Pull Request Process\n\n### Before Creating PR\n- [ ] Code is tested and working\n- [ ] All tests pass (`npm test`)\n- [ ] Code is linted (`npm run lint`)\n- [ ] No console errors or warnings\n- [ ] Branch is up to date with target branch\n\n### PR Requirements\n1. **Descriptive title** following conventional commit format\n2. **Clear description** of changes and reasoning\n3. **Link related issues** using `Closes #123` or `Fixes #123`\n4. **Screenshots** for UI changes\n5. **Test instructions** for reviewers\n\n### PR Template\n```markdown\n## Changes\n- Brief description of changes\n- What problem this solves\n\n## Type of Change\n- [ ] Bug fix\n- [ ] New feature\n- [ ] Breaking change\n- [ ] Documentation update\n\n## Testing\n- [ ] Tests pass locally\n- [ ] Manual testing completed\n- [ ] Edge cases considered\n\n## Screenshots (if applicable)\n[Add screenshots here]\n\n## Checklist\n- [ ] Code follows style guidelines\n- [ ] Self-review completed\n- [ ] Comments added for complex code\n- [ ] Documentation updated\n```\n\n### Review Guidelines\n- **At least 1 approval** required\n- **All conversations resolved** before merge\n- **No merge conflicts**\n- **CI checks passing**\n\n---\n\n## GitHub Branch Protection Rules\n\n### Main Branch Protection\n- [x] Require pull request reviews before merging\n- [x] Require status checks to pass before merging\n- [x] Code formatting\n- **ESLint** - Code linting\n- **TypeScript** - Type checking\n- **Jest** - Run tests\n\n### CI/CD Pipeline\n1. **Lint Check** - ESLint validation\n2. **Type Check** - TypeScript compilation\n3. **Test Suite** - Jest unit tests\n4. **Build Check** - Next.js build success\n5. **Security Scan** - Dependency vulnerabilities\n\n---\n\n## Development Environment Setup\n\n### Required Tools\n- **Node.js** 18+ \n- **npm** 8+\n- **Git** 2.30+\n- **VS Code** with recommended extensions\n\n### Setup Commands\n```bash\n# Clone repository\ngit clone https://github.com/vibedev-squad-official/vibedev-squad-official.git\ncd vibedev-squad-official\n\n# Install dependencies\nnpm install\n\n# Copy environment variables\ncp .env.example .env.local\n# Edit .env.local with your values\n\n# Start development server\nnpm run dev\n\n# Run tests\nnpm test\n\n# Run linting\nnpm run lint\n```\n\n### Development Scripts\n```bash\nnpm run dev          # Start development server\nnpm run build        # Build for production\nnpm run start        # Start production server\nnpm run lint         # Run ESLint\nnpm run test         # Run Jest tests\nnpm run test:watch   # Run tests in watch mode\nnpm run test:coverage # Run tests with coverage\n```\n\n---\n\n## Emergency Procedures\n\n### Hotfix Process\n1. Create `hotfix/description` branch from `main`\n2. Make minimal fix\n3. Test thoroughly\n4. Create PR to `main` (bypass normal review if critical)\n5. Merge to `main` and deploy immediately\n6. Cherry-pick fix to `develop`\n7. Document incident in changelog\n\n### Rollback Process\n1. Identify last stable commit\n2. Create rollback branch\n3. Revert problematic changes\n4. Deploy immediately\n5. Post-mortem analysis\n6. Create issue to properly fix\n\n---\n\n## Contact & Support\n\n**Development Team Lead:** [To be assigned]\n**Code Review Team:** [To be assigned]\n**DevOps/Infrastructure:** [To be assigned]\n\n**Questions?** Create an issue with the `question` label.
